# План реализации системы автопубликации и модерации контента

Этот документ описывает этапы разработки функционала для автоматической публикации постов в социальные сети (VK, Telegram) и на главную страницу сайта, включая систему модерации волонтерского контента.

## 1. Подготовительный этап (Интеграции)
- [ ] **Telegram Bot API**:
    - Создание бота через @BotFather.
    - Реализация модуля отправки сообщений и медиафайлов.
- [ ] **VK API**:
    - Создание приложения VK.
    - Получение Access Token с правами `wall` и `offline`.
    - Реализация модуля для метода `wall.post`.
- [ ] **Настройка окружения**:
    - Добавление необходимых библиотек в `requirements.txt` (`requests`, `python-dotenv`, `apscheduler`).

## 2. Backend: Логика модерации (`routes/lc.py`)
- [ ] **Роль Волонтера**:
    - Создание эндпоинта для отправки черновика поста (`POST /post/create`).
    - Установка начального статуса `pending_review`.
- [ ] **Роль Модератора**:
    - Эндпоинт для получения списка постов на проверку (`GET /moderation/list`).
    - Эндпоинт для одобрения/отклонения поста (`POST /moderation/approve/<id>`).
    - При одобрении: смена статуса на `scheduled` или `published`.

## 3. Автопубликация и Фоновые задачи
- [ ] **Планировщик (Scheduler)**:
    - Использование `APScheduler` для периодической проверки таблицы `posts`.
    - Выборка постов со статусом `scheduled`, у которых `scheduled_at <= current_time`.
- [ ] **Сервис публикации**:
    - Логика итерации по привязанным `social_accounts`.
    - Обработка ошибок публикации и обновление статуса в таблице `publications`.
    - Запись `external_post_id` для будущей аналитики.

## 4. Frontend: Интерфейс управления (`templates/lc.html`)
- [ ] **Панель волонтера**:
    - Форма создания поста (текст, загрузка изображений, выбор даты публикации).
    - Список своих постов со статусами.
- [ ] **Панель модератора**:
    - Интерфейс "Входящие" для проверки новых постов.
    - Кнопки быстрого действия: "Опубликовать сейчас", "Запланировать", "Отклонить (с комментарием)".

## 5. Главная страница (`routes/home.py` & `templates/home.html`)
- [ ] **Отображение контента**:
    - Вывод постов со статусом `published` на главной странице.
    - Сортировка по дате публикации (`published_at`).
    - Реализация пагинации для ленты новостей.

## 6. Тестирование и деплой
- [ ] Тестирование отправки в тестовые каналы/группы.
- [ ] Проверка корректности смены статусов в БД.
- [ ] Финальная сборка и проверка прав доступа (RBAC).
